# 群消息事件数据结构文档

## 概述

该数据结构描述了一个群聊消息事件（`GROUP_MESSAGE_CREATE`），包含消息正文、提及（@）信息、引用回复等。数据以网关事件格式传递，字段包括操作码、序列号、事件类型等。

## 事件根结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `op` | integer | 操作码，示例中为 `0` |
| `s` | integer | 序列号，示例中为 `4` 或 `10` |
| `t` | string | 事件类型，值为 `"GROUP_MESSAGE_CREATE"` |
| `id` | string | 事件唯一标识 |
| `d` | object | 消息数据对象（见下文） |

## 消息数据对象（`d`）

| 字段 | 类型 | 说明 |
|------|------|------|
| `author` | object | 消息发送者信息 |
| `member_openid` | string | 成员在群中的唯一标识 |
| `union_openid` | string | 用户跨应用的唯一标识 |
| `username` | string | 发送者昵称 |
| `content` | string | 消息原始内容，包含 `<@user_id>` 和 `<@all>` 等标签 |
| `group_id` | string | 群组ID |
| `group_openid` | string | 群组开放标识 |
| `id` | string | 消息ID（机器人侧） |
| `mentions` | array | 提及的用户或特殊范围列表 |
| `message_scene` | object | 消息场景信息（如引用回复） |
| `message_type` | integer | 消息类型，普通文本为 `0`，引用消息为 `103` |
| `timestamp` | string | 消息时间，ISO 8601 格式 |
| `msg_elements` | array | 当 `message_type=103` 时，包含被引用的原始消息片段 |

### `author` 对象

| 字段 | 类型 | 说明 |
|------|------|------|
| `bot` | boolean | 是否为机器人 |
| `id` | string | 发送者ID |

### `mentions` 数组元素

每个提及项表示一个被 @ 的对象，可能包含以下字段（根据上下文组合）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `bot` | boolean | 是否为机器人（仅当提及机器人时出现） |
| `id` | string | 被提及用户的ID（当 `bot` 为 true 时） |
| `is_you` | boolean | 是否指向当前机器人用户 |
| `member_openid` | string | 被提及成员的 openid |
| `scope` | string | 提及范围：`"single"`（单人）、`"all"`（全体成员） |
| `username` | string | 被提及用户的昵称 |

### `message_scene` 对象

| 字段 | 类型 | 说明 |
|------|------|------|
| `ext` | array | 扩展信息，包含 `msg_idx` 或 `ref_msg_idx` 等引用索引 |
| `source` | string | 场景来源，示例中为 `"default"` |

### `msg_elements` 数组（仅引用消息）

当 `message_type=103` 时，该数组包含被引用的原始消息信息：

| 字段 | 类型 | 说明 |
|------|------|------|
| `author` | object | 原消息发送者（含 `bot` 和 `username`） |
| `content` | string | 原消息内容（已去除 `<@...>` 标签的纯文本格式） |
| `message_type` | integer | 原消息类型，通常为 `103` 表示引用 |
| `msg_idx` | string | 原消息的引用索引 |

    "message_type": 103,
    "msg_elements": [
      {

字段备注

content 中的 <@用户ID> 与 mentions 数组对应，可用于解析具体被 @ 的对象。
msg_elements 仅在引用消息中出现，用于提供被引用消息的正文和作者。
message_scene.ext 中的 ref_msg_idx 和 msg_idx 用于关联引用的原始消息。
部分字段出现截断（如 group_openid、member_openid）为示例展示，实际应为完整字符串。
